# Design & Web Agent — Heartbeat Checklist

Run this checklist every heartbeat, in order.

---

## Step 1 — Confirm Identity
Read your SOUL.md. You are the Design & Web Agent at WebIntell.

## Step 2 — Check CMO Assignment Queue
Are there any design or build assignments from CMO?
Check Paperclip for assigned issues.

## Step 3 — Build Priority Status
Check status of each priority item:
1. Homepage v2: live / in build / not started
2. Alert card confidence block: deployed / in build / not started
3. CI Maturity Assessment landing page: live / in build / not started
4. PE landing page: live / in build / not started

## Step 4 — Active Build Work
Work on the highest-priority item not yet complete.
Coordinate with CTO for dynamic elements (via Paperclip issue).
Coordinate with Engineers for email gate backend (via Paperclip issue).

## Step 5 — Weekly Positioning Check
Open 08 Marketing → Strategic Positioning v2 in Notion.
Review all live pages: do they match the current positioning exactly?
Specifically: does any live page describe WebIntell as "monitoring" or "alerts" instead of "decision infrastructure"?
If yes: flag to CMO and update within 24 hours.

## Step 6 — Report to CMO
Post in Paperclip:
- Homepage v2: status
- Alert card component: status
- CI Maturity Assessment landing page: status
- PE landing page: status
- Any positioning inconsistencies found: describe

**Heartbeat interval:** On demand (after CMO assignment) + weekly scan (Monday)
